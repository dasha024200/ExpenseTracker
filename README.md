# ExpenseTracker - Система управління особистими витратами

## Про проект

**ExpenseTracker** - це консольна система для відслідковування та аналізу особистих видатків. Проект розроблено як частина навчального курсу з застосуванням сучасних принципів розробки ПЗ (SOLID, шарувата архітектура, тестування).

## Особливості

✓ **Додавання витрат** - Записуйте видатки з категоріями, сумами та описами  
✓ **Категоризація** - 8 вбудованих категорій витрат (Їжа, Транспорт, Розваги та інші)  
✓ **Фільтрація** - Перегляд витрат за місяцами та періодами  
✓ **Статистика** - Аналіз витрат по категоріям  
✓ **Розраховані показники** - Загальна сума, середня витрата, мінімум/максимум  
✓ **Обробка помилок** - Валідація даних та інформативні повідомлення про помилки

## Користування

При запуску програма показує головне меню з наступними опціями:

```
╔════════════════════════════════════════╗
║     Система управління витратами       ║
║         ExpenseTracker v1.0            ║
╚════════════════════════════════════════╝

Головне меню:
1. Додати витрату
2. Переглянути витрати за місяц
3. Переглянути статистику по категоріям
4. Переглянути всі витрати
5. Переглянути статистику
0. Вихід
```

### Приклад використання

#### Додавання витрати

1. Виберіть опцію "1"
2. Виберіть категорію з доступного списку
3. Введіть суму в гривнях
4. Введіть опис витрати

#### Перегляд статистики

1. Виберіть опцію "3" для статистики по категоріям
2. Система покаже розподіл витрат за кожною категорією

## Архітектура

Проект побудований за принципом **шарування** з чітким розділенням відповідальності:

### Domain Layer (ExpenseTracker.Domain)
- Сутності: `Expense`, `Category`
- Бізнес-правила: валідація, інваріанти
- Контракти: інтерфейси `IExpenseRepository`, `ICategoryRepository`

### Application Layer (ExpenseTracker.Application)
- `ExpenseService` - орхестрація операцій
- Залежність від Domain
- LINQ-запити для обробки даних

### Infrastructure Layer (ExpenseTracker.Infrastructure)
- Реалізація репозиторіїв
- Зберігання даних (поки в памяті)

### Console Layer (ExpenseTracker.Console)
- Користувацький інтерфейс
- Обробка команд

### SOLID принципи

- **S**ingle Responsibility: кожен клас має одну причину для змін
- **O**pen/Closed: добавляю нову функціональність без зміни існуючого кода
- **L**iskov Substitution: репозиторії могли бути замінені
- **I**nterface Segregation: маленькі фокусовані інтерфейси
- **D**ependency Inversion: залежність від абстракцій (інтерфейсів)

   - Успішне створення витрати

   - Розрахунок середньої витрати
