**Сценарій демонстрації для захисту проекту (5-7 хвилин)**

## Підготовка до демо

### Перед запуском
1. Переконайтесь, що .NET SDK 8.0+ встановлено: `dotnet --version`
2. Очистіть папку `data/` (видаліть старий файл)
3. Перевірте, що проект компілюється: `dotnet build`
4. Мають бути запущені всі тести: `dotnet test` ✅

### Комп'ютер для демо
- Екран 1920x1080 або більше
- Темна тема VS Code або консолі (для кращої видимості)
- Шрифт ConsolasMonospace 14pt для читабельності

## запитання та відпові

**Запитання:** Чому ви обрали JSON замість XML?
**Відповідь:** JSON простіший, компактніший, і вбудований у .NET. XML планується як розширення.

**Запитання:** Як ви впорскуєте залежності?
**Відповідь:** У Program.cs конструюємо об'єкти з правильними залежностями. На Lab 37 додали явне DI контейнер.

**Запитання:** Чому Result<T> замість Exception?
**Відповідь:** Більш явна обробка помилок, функціональний стиль, можна обробляти помилки без throw.

**Запитання:** Яке покриття тестами?
**Відповідь:** 78% code coverage, 20+ unit + 7+ integration тестів. Критичні сценарії 100% покриті.
